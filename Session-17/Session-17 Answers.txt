1. Δεν μπορούμε να κάνουμε κάποιο HTTP request χωρίς να γνωρίζουμε ολόκληρο το URL του site/server.
Αν γνωρίζουμε το base URL, τη δομή και το format, θα μπορούσαμε να προσπαθήσουμε να δημιουργήσουμε το full URL με τη χρήση variables.

2. Το content-type του response θα είναι: image/jpeg

3. Χρησιμοποιώντας ως header: 
	Cache-Control: max-age = 86 400 (πόσα seconds αντιστοιχούν σε μια μέρα).
   	ή 
	Expires: Thu, 2 Feb 2023 13:00:00 GMT (example) για να ορίσω συγκεκριμένη ημερομηνία-ώρα.

4. Ναι, μπορώ να έχω μια εφαρμογή που κάνει μόνο GET Requests. Θα εξυπηρετούσε μόνο τη συλλογή πληροφοριών από ένα σέρβερ.

5. Ναι, μπορώ να έχω μια εφαρμογή που κάνει μόνο POST Requests. Θα εξυπηρετούσε μόνο την αποστολή δεδομένων σε ένα σέρβερ.

6. To Status Code που θα εμφανιζόταν θα ήταν: 400 Bad Request μιας και πρόκειται για syntax error απο την πλευρά του client

7. To status code σε μια τέτοια περίπτωση πιθανότατα θα ήταν: 500 internal server error.

8. Θα μπορούσαμε να κάνουμε Login με GET Request, ωστόσο για λόγους ασφάλειας είναι καταληλλότερη η χρήση του POST request.

9. Η κύρια διαφορά μεταξύ των μεθόδων PUT & POST είναι ότι η PUT χρησημοποιείται για την ενημέρωση πόρων στον σέρβερ, ενώ η POST χρησημοποιείται για τη δημιουργία νέων πόρων στον σερβερ. Δηλαδή, πολλαπλές εκτελέσεις μιας αμετάβλητης εντολής PUT θα έχουν το ίδιο αποτέλεσμα με μια μόνο εκτέλεση της ίδιας εντολής, σε αντίθεση με την εντολή POST, η οποία δημιουργεί νέους πόρους στον σέρβερ με κάθε της εκτέλεση.   

10. Για να ανεβάσουμε ένα αρχείο (εκτός και αν αντικαθιστούμε κάποιο υπάρχον) είναι καταλληλότερη η χρήση της μεθόδου POST.
